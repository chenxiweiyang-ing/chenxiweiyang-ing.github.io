webpack定义

webpack是一个模块打包工具。(最初只对js打包，现在可对任意文件打包)

webpack对ES Module模块引入方式、CommonJS模块引入规范、CMD、AMD都能正确识别。

CommonJS模式

webpack 安装

1.安装node。默认也安装了npm

2.进入要安装的文件夹运行npm init生成package.json文件

3.运行npm install webpack webpack-cli

若安装在项目内，可能运行webpack命令报错，可使用npx，如npx webpack -v。

npm info webpack可查看webpack版本号 可用npm install webpack@版本号

文件名称：webpack.config.js

loader

webpack不能识别非js的文件打包，所以需要添加loader，使用看配置文件。
1.打包静态资源

url-loader与file-loader类似，只是前者打包后的文件会放在打包后的js文件里，比如图片会变为base64.可配置limit参数。
2.打包样式（具体可看webpack的官方文档）
‘style-loader’,’css-loader’,’sass-loader’

loader 的执行顺序是从下到上，从右到左的执行顺序。

plugins

在webpack运行到某个时刻的时候，帮你做一些事情

1.HtmlWebpackPlugin(代码参考上边)
会在打包结束后，自动生成一个html文件，并把打包生成的js自动引入到html中
2.cleanWebpackPlugin

打包之前清理某个文件。

安装：npm install clean-webpack-plugin -d

entry

会打包两次，生成main.js,sub.js两个文件

sourceMap 具体参考webpack文档

它是一个映射关系，若代码出错，可映射到源代码的哪个文件哪行出错。
devTool:’none’ 可关掉sourceMap
devTool:’source-map’开启sourceMap，会多出XX.js.map文件
devTool:’inline-source-map’开启sourceMap，映射文件会在main.js里
devTool:’cheap-inline-source-map’同上，报错提示直到第几行，打包速度会加快
开发环境建议：cheap-module-eval-source-map
生产环境建议：cheap-module-source-map提示效果较好
